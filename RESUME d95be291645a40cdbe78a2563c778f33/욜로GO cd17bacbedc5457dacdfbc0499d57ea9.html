🏀트러블 슈팅

1. CORS 설정

  • 문제상황
    • 프런트 서버와 연결 시 CORS 연결 오류가 뜸

  • 시도한 방법
    • Spring Security를 사용하여 전체 주소 허용
      • 브라우저 내에서 이러한 설정을 허용하지 않아 오류가 지속적으로 생김

  • 해결한 방법

    WebConfiguration 설정을 통하여 CORS에서 프런트 서버를 허용 - 추가적인 불안전 요청 처리(PUT, DELETE, PATCH)를 위해 AllowedMethod 재설정

2. JPA N+1 쿼리 성능개선

  • 문제상황
    • 게시물을 조회하는 API 실행 시, DB에 쿼리가 45번이 요청되는 현상 발생.

  • 시도한 방법
    • QueryDSL 을 사용하여 쿼리 수 조정
      • 숙련도 문제로 사용 실패
    • JPA의 fetch join을 사용하여 쿼리 수 조정

  • 해결한 방법

    @Query 문을 사용하여 Entity를 fetch join 사용하여 연결, 45번에서 1번으로 축소

3. JWT Refresh Token 기능 추가

  • 문제상황
    • 이전에 사용되는 JWT 방식이 RefreshToken으로 권한을 확인하는 방식으로 잘못됨.

  • 구현한 방법
    • 로그인 시 AccessToken과 RefreshToken을 발급하고 RefreshToken은 레포지코리에 저장을 해두어 로그아웃 시에 두 토큰 다 만료시키도록 설정
    • RefreshTokenAPI 실행 시 클라이언트가 소지한 RRefreshToken이 유효할 시, RefreshToken요청자가 레포지토리에 저장된 값과 같을 시, AccessToken을 재발급 하는 방식으로 수정.

4. 배포시 파일 업로드 경로 오류, 권한 설정 오류

  • 문제상황
    • 다른 주소에서 파일을 업로드 할 시 배포한 사람 외에 파일 업로드가 실패하는 현상, 혹은 경로가 잘못되어 파일이 올라가지 않는 오류.

  • 시도한 방법
    • 파일 경로를 "image/"로 바꿔 jar파일이 있는 곳에 "image"폴더를 생성
    • EC2 권한을 모든 포트에서 가능하도록 지역과 포트 설정을 하여 해결

  • 해결한 방법
